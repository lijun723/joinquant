## 一、核心思想

**量化交易的本质：** 用系统化的方法替代主观判断。黑箱不是神秘，而是"输入数据→策略逻辑→输出交易信号"的可重复流程。

```
市场数据 → [策略模型] → 交易信号 → 下单执行 → 风控监控
                          ↑
                    可重复、可验证、可改进
```

**量化 vs 主观交易：**

| 维度 | 主观交易 | 量化交易 |
|------|---------|---------|
| 决策依据 | 经验/直觉/消息 | 数据/统计/模型 |
| 一致性 | 情绪干扰大 | 严格执行规则 |
| 可回溯 | 难复盘（记不清当时想法） | 完全可回放 |
| 改进方式 | 靠经验积累 | 靠数据回测验证 |
| 规模上限 | 一个人盯几只 | 全市场几千只 |

## 二、量化系统的六大模块

作者把量化系统拆成6个核心模块，每个模块独立可替换：

```
                   ┌─────────────┐
                   │   Alpha 模型  │  ← 产生交易信号（核心）
                   └──────┬──────┘
                          ↓
                   ┌─────────────┐
                   │  风控模型  │  ← 过滤掉不能下单的信号
                   └──────┬──────┘
                          ↓
                   ┌─────────────┐
                   │  交易成本模型 │  ← 估算滑点/手续费
                   └──────┬──────┘
                          ↓
                   ┌─────────────┐
                   │  执行模型   │  ← 具体怎么下单
                   └──────┬──────┘
                          ↓
                   ┌─────────────┐
                   │  投资组合模型│  ← 仓位分配
                   └──────┬──────┘
                          ↓
                   ┌─────────────┐
                   │  数据模型   │  ← 输入数据的清洗/对齐
                   └─────────────┘
```

### 模块1：Alpha 模型（预测模型）

**核心问题：** 什么样的股票/期货会涨？

三种主要策略类型：

#### A. 趋势跟踪（最大众）
- **原理：** 过去涨的还会涨，过去跌的还会跌
- **时间框架：** 日内到数月不等
- **经典实现：** 双均线、唐奇安通道突破、MACD
- **适用环境：** 趋势行情（单边牛/熊）
- **致命弱点：** 震荡市反复打脸（双杀）
- **A股案例：** 2019年白马股趋势、2020年新能源趋势

#### B. 均值回归（反趋势）
- **原理：** 涨多了会跌，跌多了会涨（价格围绕价值波动）
- **经典实现：** 布林带、RSI超买超卖、配对交易
- **适用环境：** 震荡行情
- **致命弱点：** 强趋势中逆势交易会亏很惨
- **A股案例：** 大部分时间是震荡市，均值回归策略胜率高但单笔收益小

#### C. 因子模型（基本面量化）
- **原理：** 某些属性（低PE、高ROE、低波动）能预测未来收益
- **经典实现：** 多因子打分、IC分析、分层回测
- **常见因子：**
  | 因子 | 方向 | A股有效性 |
  |------|------|----------|
  | 估值（PE/PB） | 越低越好 | 有效，但时间尺度长 |
  | 质量（ROE） | 越高越好 | 中期有效 |
  | 动量（过去收益） | 越高越好 | 短期有效，长期反转 |
  | 低波动 | 越低越好 | A股很有效（低波动异象） |
  | 换手率 | 越低越好 | 高换手=散户扎堆=负收益 |

**作者的核心观点：** 没有永远的圣杯。每种策略都有其**盈利环境**。最重要的不是找到完美的策略，而是**知道当前是什么市场，用对应的策略**。

### 模块2：风控模型

**风控不是锦上添花，是生存底线。**

#### 三种风控层次：

```python
# 层级1：单笔止损（硬止损）
stop_loss = entry_price * 0.92  # -8%无条件割

# 层级2：移动止损（ATR跟随）
atr = calculate_atr(20)
trailing_stop = max(current_highs[-20:]) - 2.5 * atr

# 层级3：组合止损（总回撤控制）
if portfolio.returns < -0.15:  # 总回撤超15%
    liquidate_all()
```

#### 作者强调的两个关键概念：

1. **路径依赖** — "破产式亏损"：一次大亏可能让你永远退出市场
   - 回撤50%需要100%的收益才能回来
   - 止损的意义不是减少亏损，而是**保证你不会一次出局**

2. **滑点是最大的隐藏成本** — 回测里的完美成交价和实际成交价之间的差距
   - 流动性差的股票滑点可能是手续费的好几倍
   - 回测时必须模拟滑点，否则实盘绩效会大幅缩水

### 模块3：交易成本模型

**很多人亏钱不是因为策略不好，是因为成本太高。**

```
总成本 = 佣金 + 印花税 + 滑点 + 冲击成本
```

**实例计算：**
```python
commission = 0.00025  # 万2.5
stamp_tax = 0.001     # 千1（卖出）
slippage = 0.001      # 千1 滑点估算

# 一次买入卖出的总成本
buy_cost = 100000 * (commission + slippage)          # ≈ 125 元
sell_cost = 100000 * (commission + stamp_tax + slippage)  # ≈ 225 元
round_trip = buy_cost + sell_cost                    # ≈ 350 元 (0.35%)

# 如果一个策略年交易50次，成本吃掉17.5%
# 如果你的策略年化收益只有15%，那实盘就是亏钱的！
```

**关键结论：** 高频策略在A股很难跑赢——印花税太高了。

### 模块4：执行模型

**如何把信号变成成交单。**

| 执行方式 | 适用 | 风险 |
|---------|------|------|
| 市价单 | 流动性好、要快速成交 | 滑点大 |
| 限价单 | 想控制价格 | 可能不成交 |
| TWAP | 大单分批执行 | 算法复杂 |
| VWAP | 追求市场均价 | 需要接入券商API |

**A股实际：** 个人量化大多用**限价单**，模拟盘用市价单。实盘时挂限价单等成交，不成交就撤单重挂。

---

### 模块5：投资组合模型

**不要把所有资金押在同一只股票上。**

核心原则：
- **分散化**：5-20只股票，不同行业
- **仓位管理**：凯利公式或固定比例
- **再平衡**：定期调整到目标权重

**凯利公式简要版：**
```python
f = (p * b - q) / b
# f = 最优仓位比例
# p = 胜率
# q = 1 - p
# b = 盈亏比（平均盈利 / 平均亏损）

# 示例：胜率45%，盈亏比2:1
f = (0.45 * 2 - 0.55) / 2 = 0.175  # 17.5%
# 实际建议用 1/4 凯利 → 4.4%
# 凯利比例其实很激进，1/4凯利更安全
```

### 模块6：数据模型

**垃圾进，垃圾出（GIGO）。**

常见数据问题：
1. **幸存者偏差** — 回测只用了现在的成分股，退市的没算进去
2. **前视偏差** — 用到了当时还不知道的数据（如财报发布前就用了最新数据）
3. **未来函数** — 计算中用到了未来的价格/数据
4. **数据对齐** — 不同数据源的时间戳不一致

**A股特别注意：**
- 用 `get_price` 时加上 `skip_paused=True`
- 用 `attribute_history` 确保是策略运行时点
- ST/退市股票必须过滤

## 三、策略开发流程（作者推荐的套路）

```
Step 1: 从直觉出发
  找一个你相信的市场规律（"低PE的股票长期跑赢"）
  
Step 2: 验证直觉
  用数据检验：IC分析 / 分层回测 / t检验
  
Step 3: 实现规则
  把直觉变成可执行的代码规则
  
Step 4: 样本内回测
  用70%的数据跑回测，调参数
  
Step 5: 样本外验证
  用30%的"没看过"的数据跑一次，如果结果好很多→过拟合
  
Step 6: 压力测试
  放在最极端的行情（2015股灾、2020疫情）跑
  
Step 7: 模拟盘
  实盘跑之前，至少跑1个月以上的模拟盘
  
Step 8: 小实盘
  用最小资金（5,000元）跑实盘
  
Step 9: 评估+迭代
  实盘和回测的差异在哪？→ 改进
```

**作者特别强调：**
> Step 5 是最容易被跳过的。很多人 Step 4 跑出好结果就直接飞到 Step 8 了。
> 这就像考试：你用往年真题练了一百遍，考了满分，然后去参加同一场考试——这不叫水平高。

## 四、如何评估一个策略（回测报告解读）

**必看指标（按重要性排序）：**

| 指标 | 好 | 一般 | 差 | 含义 |
|------|----|------|-----|------|
| **年化收益** | >15% | 5-15% | <5% | 每年赚多少 |
| **最大回撤** | <10% | 10-20% | >20% | 最多亏多少 |
| **夏普比率** | >2 | 1-2 | <1 | 每承担1单位风险赚多少 |
| **胜率** | >50% | 30-50% | <30% | 赚钱交易占比 |
| **交易次数** | >50次/年 | — | 太少 | 统计意义够不够 |
| **年化波动率** | — | 15-25% | >30% | 策略波动大不大 |

**作者的重点：夏普比率 > 年化收益**

```
夏普 2.0 年化20% vs 夏普 0.5 年化30%
前者更值得信任——因为前者收益的"质量"更高
```

**另一个关键指标：收益来源分析**
- 哪几笔交易贡献了大部分利润？
- 如果去掉最好的3笔交易，策略还赚钱吗？
- 如果去掉最差的3笔交易呢？

## 五、量化团队的构成（了解即可）

如果你将来想组建团队，需要这几类人：

| 角色 | 做什么 |
|------|--------|
| 策略研究员 | 想点子、做回测 |
| 数据工程师 | 清洗数据、搭建数据管道 |
| 软件工程师 | 写交易系统、回测框架 |
| 风控主管 | 监控风险、设置限额 |
| 交易员 | 监控执行、处理异常 |

**作者吐槽：** 很多人以为量化就是"写个程序自动交易"，
实际上80%的时间花在数据处理和策略迭代上，真正写交易代码的时间不到10%。

## 六、常见误区总结

| # | 误区 | 真相 |
|---|------|------|
| 1 | 量化就是写个程序自动交易 | 量化是系统化决策，自动交易只是执行环节 |
| 2 | 回测好就能实盘赚钱 | 回测和实盘之间有巨大鸿沟（滑点/心理/数据延迟） |
| 3 | 策略越复杂越好 | 很多最简单的策略（双均线/估值）长期表现并不差 |
| 4 | 参数调得越细越好 | 这是过拟合，样本外大概率翻车 |
| 5 | 波动大的策略收益高 | 风险调整后的收益（夏普）才是关键 |
| 6 | 胜率很重要 | 很多赚钱的策略胜率只有30-40%，但盈亏比大 |
| 7 | 量化不需要懂市场 | 策略的底层逻辑必须对市场有深刻理解 |

## 七、核心金句摘录

1. **"交易系统的终极测试不是它在样本内表现如何，而是它在样本外表现如何。"**

2. **"如果你问十个量化交易员他们的关键风险是什么，九个会说是'模型失效'。"**
   → 所有策略最终都会失效，关键是你能不能在它失效时及时发现并调整。

3. **"回撤控制是量化交易的核心竞争力。不是因为你能避免亏损，而是因为你能在市场最疯狂的时候保持理性。"**

4. **"量化交易不是寻找圣杯，而是建立一个可以持续改进的系统。"**

5. **"滑点是你最沉默的敌人。它不会出现在你的回测报告里，但会出现在你的实盘账户里。"**
