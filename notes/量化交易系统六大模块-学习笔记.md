# 1. 数据模型

> **核心作用**：把原始数据变成策略可用的干净的、对齐的、无偏的时间序列

### 1.1 数据模型要解决什么问题

| 问题 | 说明 | 后果 |
|------|------|------|
| **幸存者偏差** | 只收录现在还在的股票，退市的不见了 | 回测虚高，实际会踩雷 |
| **前视偏差** | 用到了未来才知道的信息 | 回测完美，实盘血亏 |
| **复权不一致** | 分红送股后价格跳跃 | 收益率计算错误 |
| **时间对齐** | 不同数据源的 OHLC 时间戳不同 | 信号和价格错位 |
| **缺失值** | 停牌、节假日、数据源中断 | NaN 传播崩溃 |

#### 步骤 B：数据清洗与对齐

```python
def clean_align(df_list, tickers, freq='D'):
    """
    对齐多只股票的时间序列
    参数:
        df_list: [df1, df2, ...] 每只股票的 DataFrame
        tickers: ['000001.XSHE', ...]
        freq: 'D' 日线, 'W' 周线
    """
    prices = pd.DataFrame()
    for df, ticker in zip(df_list, tickers):
        # 去重、排序、填补
        df = df.drop_duplicates(subset=['date'])
        df = df.sort_values('date')
        df = df.set_index('date')
        
        # 只保留收盘价
        s = df['close'].astype(float)
        s.name = ticker
        
        # 停牌期间 NaN 前向填充（用前一天的收盘价代替）
        s = s.fillna(method='ffill', limit=20)
        
        prices = pd.concat([prices, s], axis=1)
    
    # 重采样对齐
    prices = prices.resample(freq).last()
    
    # 剔除上市不满 60 天的股票
    min_days = 60
    prices = prices.loc[:, prices.count() >= min_days]
    
    return prices

# ✅ 用 3-5 只股票测试，打印 prices.head() 应无 NaN
```

# 2. 阿尔法模型

> **核心作用**：产生交易信号——什么时候买、什么时候卖

### 2.1 阿尔法的三种流派

| 流派 | 思路 | 典型信号 | 持有期 |
|------|------|----------|--------|
| **趋势跟踪** | 强者恒强 | 突破N日高点、均线上穿 | 中长线（天级） |
| **均值回复** | 涨多会跌 | RSI > 70 卖出、布林带回归 | 短线（小时级） |
| **因子选股** | 找优质公司 | 低PE+高ROE+低波动 | 月级调仓 |

### 2.2 信号生成的通用框架

```
原始数据 → 因子计算 → 信号转换 → 叠加/加权 → 最终信号
                                ↓
                          可选：机器学习
```

#### 示例 A：趋势跟踪（唐奇安突破 — 你已在用的策略）

```python
def donchian_signal(close, period=20):
    """唐奇安通道突破信号"""
    high = close.rolling(period).max()
    low = close.rolling(period).min()
    
    # 突破上轨 = 买入信号
    buy = close > high.shift(1)
    # 跌破下轨 = 卖出信号
    sell = close < low.shift(1)
    
    return buy.astype(int) - sell.astype(int)  # 1=买入, -1=卖出, 0=持仓

close = pd.Series([...])  # 你的价格数据
signals = donchian_signal(close)

# ✅ 打印最后 10 行信号，应看到 0/1/-1
```

#### 示例 B：双均线交叉（动量信号）

```python
def ma_crossover(close, fast=5, slow=20):
    """双均线金叉死叉信号"""
    ma_fast = close.rolling(fast).mean()
    ma_slow = close.rolling(slow).mean()
    
    # 金叉 = 快线上穿慢线
    crossover = (ma_fast > ma_slow) & (ma_fast.shift(1) <= ma_slow.shift(1))
    # 死叉 = 快线下穿慢线
    crossunder = (ma_fast < ma_slow) & (ma_fast.shift(1) >= ma_slow.shift(1))
    
    signal = pd.Series(0, index=close.index)
    signal[crossover] = 1
    signal[crossunder] = -1
    signal = signal.replace(0, method='ffill')  # 持仓延续
    
    return signal

# ✅ 在金叉日看到 1，死叉日看到 -1
```

#### 示例 C：多因子信号组合

```python
def multi_factor_signal(close, volume):
    """动量 + 量能双因子组合"""
    # 因子1：20日动量
    momentum = close.pct_change(20)
    signal_mom = (momentum > 0).astype(float)
    
    # 因子2：量比（当日量/20日均量）
    vol_ratio = volume / volume.rolling(20).mean()
    signal_vol = (vol_ratio > 1.5).astype(float)
    
    # 等权组合
    signal = (signal_mom * 0.5 + signal_vol * 0.5)
    
    # 只有同时满足两个条件才开仓（保守模式）
    signal_conservative = (signal_mom & signal_vol).astype(int)
    
    return signal_conservative

# ✅ 只有动量+量能同时确认时才产生信号
```

### 2.3 阿尔法模型的评价指标

| 指标 | 公式 | 含义 |
|------|------|------|
| IC | 因子值与未来收益的秩相关系数 | 预测能力。IC>0.05 算好 |
| IR | IC均值 / IC标准差 | 稳定性。IR>0.5 优秀 |
| 换手率 | 日均信号变化比例 | 调仓成本。越低越好 |

# 3. 风险模型

> **核心作用**：过滤掉那些看起来赚钱但实际不能做的交易

### 3.1 你不该忽略的四大风险

| 风险类型 | 具体含义 | 量化处理 |
|----------|----------|----------|
| **市场风险** | 大盘暴跌时系统回撤 | Beta 对冲、止损线 |
| **行业风险** | 某个行业板块性下跌 | 行业持仓上限约束 |
| **风格风险** | 小盘/成长等风格轮动 | 风格因子暴露控制 |
| **集中度风险** | 重仓单一股票 | 单票仓位上限 |

### 3.2 风险过滤实战

#### 示例 A：大盘环境过滤（你已在用的 MA20 过滤）

```python
def market_filter(close_index, signal, threshold=0):
    """
    大盘牛熊过滤
    close_index: 沪深300收盘价序列
    signal: 个股原始信号
    threshold: MA20 斜率阈值（>0 表示上升趋势）
    """
    ma20 = close_index.rolling(20).mean()
    # 大盘在 MA20 之上=牛市可做，之下不可做(熊市不做多)
    bull = close_index > ma20
    
    filtered = signal.copy()
    # 熊市多头信号归零（只做空头或空仓）
    filtered[~bull] = 0  
    
    return filtered

# ✅ 在 2022 年熊市行情中，大部分多头信号被过滤，空仓天数明显增多
```

#### 示例 B：止损过滤（你已在用的 ATR 止损）

```python
def atr_stop(high, low, close, period=14, multiplier=2.5):
    """
    ATR 通道止损
    多头止损 = 前一日收盘 - ATR * 2.5
    """
    tr = pd.concat([
        high - low,
        abs(high - close.shift(1)),
        abs(low - close.shift(1))
    ], axis=1).max(axis=1)
    atr = tr.rolling(period).mean()
    
    stop_level = close.shift(1) - atr * multiplier
    
    # 跌破止损 = 信号归零
    return stop_level  # 与最新价比较，跌破即止损

# ✅ ATR 值越大，止损宽度越大（波动大的股票容忍度更高），反之亦然
```

#### 示例 C：Barra 风格因子风险简化版

```python
def simple_risk_constraints(positions, industry_map, max_ind_weight=0.3, max_stock_weight=0.15):
    """
    简化版风险约束
    positions: {ticker: weight} 如 {"000001.XSHE": 0.2}
    industry_map: {ticker: industry} 如 {"000001.XSHE": "金融"}
    """
    # 行业集中度检查
    ind_weights = {}
    for ticker, w in positions.items():
        ind = industry_map.get(ticker, "其他")
        ind_weights[ind] = ind_weights.get(ind, 0) + w
    
    for ind, w in ind_weights.items():
        if w > max_ind_weight:
            print(f"⚠️ {ind} 行业权重 {w:.1%} 超过限制 {max_ind_weight:.0%}")
    
    # 个股集中度检查
    for ticker, w in positions.items():
        if w > max_stock_weight:
            print(f"⚠️ {ticker} 权重 {w:.1%} 超过限制 {max_stock_weight:.0%}")
    
    return all(w <= max_ind_weight for w in ind_weights.values()) \
           and all(w <= max_stock_weight for w in positions.values())

# 测试
risk_ok = simple_risk_constraints(
    {"000001.XSHE": 0.5, "000002.XSHE": 0.3, "000003.XSHE": 0.2},
    {"000001.XSHE": "金融", "000002.XSHE": "地产", "000003.XSHE": "地产"}
)
# ✅ 应看到 000001.XSHE 超限警告
```

# 4. 成本模型

> **核心作用**：在回测中预估真实交易成本，避免纸上富贵

### 4.1 成本的三个层次

```
        佣金（固定）
交易成本 ├─ 印花税（卖出时收，A股0.05%）
         ├─ 滑点（买卖价差 + 市场冲击）
         └─ 冲击成本（大单影响）
```

**关键认识**：滑点 > 佣金，大资金时冲击 > 滑点

### 4.2 成本估算方法

#### 示例 A：JoinQuant 标准设置 → 你可以在回测中直接配置

```
# JoinQuant 回测设置
commission: 0.0003     # 万分之三（含规费）
min_commission: 5      # 最低5元
slippage_fixed: 0.01   # 固定滑点0.01元（买入时多付1分）
stamp_duty: 0.001      # 印花税千分之一（卖出时）
```

#### 示例 B：动态滑点模型

```python
def estimate_slippage(volume, avg_volume, price, trade_size_ratio=0.1):
    """
    按流动性的动态滑点估算
    volume: 当日成交量
    avg_volume: 20日均量
    price: 当前价格
    trade_size_ratio: 你的交易量占比
    """
    # 流动性比率（越接近1越好）
    liquidity = min(1.0, volume / avg_volume)
    
    # 滑点 = 买卖价差 + 冲击成本
    # 低流动性时滑点放大
    spread = 0.001  # 基准价差0.1%
    impact = trade_size_ratio * 0.01 / max(liquidity, 0.1)
    
    total_slippage = spread + impact
    return total_slippage * price

# 示例
price = 50.0
volume_high = 50_000_000  # 5000万股
volume_low = 2_000_000    # 200万股
avg_vol = 10_000_000

slippage_high = estimate_slippage(volume_high, avg_vol, price)
slippage_low = estimate_slippage(volume_low, avg_vol, price)
print(f"高流动性滑点: {slippage_high:.4f} 元")  # 约0.05-0.06
print(f"低流动性滑点: {slippage_low:.4f} 元")   # 会显著更大

# ✅ 看到低流动性股票的滑点明显大于高流动性股票
```

#### 示例 C：回测中对比有无成本

```python
def backtest_with_costs(close, signals, commission=0.0003, slippage=0.001):
    """
    带成本的简单回测
    返回有无成本的净值对比
    """
    returns = close.pct_change()
    strategy_returns = signals.shift(1) * returns  # T+1 买卖
    
    # 交易次数计算（信号变化 = 交易）
    trades = (signals != signals.shift(1)).sum()
    trade_cost = trades * (commission * 2 + slippage * 2)  # 买卖各一次
    
    # 无成本净值
    nav_no_cost = (1 + strategy_returns).cumprod()
    
    # 有成本净值（每天按百分比扣）
    daily_cost = (commission + slippage) * (signals != signals.shift(1)).astype(float)
    strategy_with_costs = strategy_returns - daily_cost
    nav_with_cost = (1 + strategy_with_costs).cumprod()
    
    return nav_no_cost, nav_with_cost, trades

# ✅ 高频策略(buy 100次)应该看到两条曲线明显分离，低频策略(buy 10次)几乎重合
```

| 日内高频 | 500+倍 | 极大 | 成本是主要敌人 |

# 5. 投资组合模型

> **核心作用**：决定了信号产生的方向下，具体该买多少

### 5.1 仓位分配的核心命题

```
信号说"买" → 买多少？
           ├─ 等权分配（简单）
           ├─ 风险平价（中等）
           ├─ 均值-方差优化（复杂）
           └─ 凯利公式（激进）
```

### 5.2 四种仓位分配方法

#### 示例 A：等权分配（最常用）

```python
def equal_weight(signals_df):
    """
    等权分配：看多的票等分资金
    signals_df: DataFrame, 每列一只股票, 值1/0/-1
    """
    longs = signals_df == 1
    n_long = longs.sum(axis=1)  # 每日持仓数
    
    weights = pd.DataFrame(0.0, index=signals_df.index, columns=signals_df.columns)
    for date in signals_df.index:
        if n_long[date] > 0:
            weights.loc[date] = longs.loc[date].astype(float) / n_long[date]
    
    return weights

# ✅ 买入5只股票时，每只权重 20%
```

#### 示例 B：波动率倒数加权（风险平价简化版）

```python
def vol_weighted(signals_df, returns_df, lookback=60):
    """
    波动率倒数加权：低波动的多分配，高波动的少分配
    """
    vol = returns_df.rolling(lookback).std()
    inv_vol = 1 / vol
    
    weights = pd.DataFrame(0.0, index=signals_df.index, columns=signals_df.columns)
    for date in signals_df.index:
        mask = signals_df.loc[date] == 1
        if mask.sum() == 0:
            continue
        
        # 取波动率倒数
        w = inv_vol.loc[date, mask]
        # 归一化
        w = w / w.sum()
        weights.loc[date, mask] = w
    
    return weights

# ✅ 银行股(波动小)权重 > 科技股(波动大)权重
```

#### 示例 C：凯利公式（最优增长）

```python
def kelly_fraction(win_rate, avg_win, avg_loss):
    """
    凯利公式：f* = (p*b - q) / b
    p = 胜率, b = 盈亏比(avg_win/avg_loss), q = 1-p
    """
    b = avg_win / avg_loss if avg_loss != 0 else 0
    f = (win_rate * b - (1 - win_rate)) / b if b != 0 else 0
    return max(0, min(f, 0.25))  # 限制最高25%

# 实战：你的策略有60%胜率，平均赚2%，平均亏1%
f = kelly_fraction(0.60, 0.02, 0.01)
print(f"凯利比例: {f:.2%}")

# ⚠️ 经验法则：实盘用半凯利（f/2），更稳健
print(f"半凯利: {f/2:.2%}")

# ✅ 凯利比例约 30%，半凯利约 15%
```

### 5.3 再平衡

```
初始分配 → 价格变化 → 权重偏离 → 再平衡 → 回归目标
                                      ↓
时间触发（每月固定日期）或 偏离触发（偏离>5%）
```

```python
def rebalance_check(weights_current, weights_target, threshold=0.05):
    """
    检查是否需再平衡
    """
    deviation = (weights_current - weights_target).abs()
    needs_rebalance = deviation.max() > threshold
    return needs_rebalance, deviation.idxmax()

# ✅ 当某个票权重从20%涨到28%（偏离8%>5%），触发再平衡
```

# 6. 执行模型

> **核心作用**：把"想买10000股平安银行"变成真实成交的单子，且价格好、不暴露意图

### 6.1 执行的关键矛盾

```
想买的量  ×  市场流动性  =  执行难度
                           ↓
                    大单→市场冲击→成本飙升→收益被吃掉
```

### 6.2 执行算法的进化

```
市价单（最简单）
   ↓ 滑点太大
限价单（等被动成交）
   ↓ 可能不成交
TWAP（时间均匀拆分）
   ↓ 容易被识别
VWAP（按成交量分布拆分）
   ↓ 最常用
冰山订单（只露出一小部分）
   ↓ 机构级
自适应算法（根据市场情况动态调整）
```

#### 示例 A：市价单 vs 限价单

```python
# 模拟两种下单方式
def simulate_execution(price, order_type='market', limit_price=None):
    """
    模拟执行结果
    market: 按当时市价成交（可能滑点）
    limit: 按限价成交（可能不成交）
    """
    if order_type == 'market':
        # 假设滑点0.1%
        executed_price = price * (1 + 0.001)
        filled = True
    else:  # limit
        if limit_price >= price:  # 买入限价高于市价
            executed_price = limit_price
            filled = True
        else:
            executed_price = None
            filled = False
    
    return filled, executed_price

# ✅ 市价单一定成交但价格差；限价单价格好但可能买不到
```

#### 示例 B：TWAP 拆分（时间加权平均价格）

```python
def twap_schedule(total_shares, trading_minutes=240, slices=24):
    """
    TWAP：等时间拆分
    total_shares: 总股数
    trading_minutes: 总交易时间（分钟）
    slices: 拆分次数
    """
    per_slice = total_shares // slices
    interval = trading_minutes // slices
    
    schedule = []
    for i in range(slices):
        schedule.append({
            'time': i * interval,
            'shares': per_slice,
            'cumulative': per_slice * (i + 1)
        })
    
    return pd.DataFrame(schedule)

# 买入10万股，拆分24次
twap = twap_schedule(100_000)
print(twap.head())
# ✅ 每次约4166股
```

#### 示例 C：VWAP 拆分（按成交量分布）

```python
def vwap_schedule(total_shares, volume_profile):
    """
    VWAP：按历史成交量分布拆分
    volume_profile: [9:30, 9:31, ...] 每分钟成交量占比
    
    实盘经验：A股早盘成交量集中在9:30-10:00，尾盘集中在14:30-15:00
    """
    # 9:30-15:00 共 240 分钟
    # 典型 A 股 Profile（前30分钟活跃，尾盘活跃）
    weights = volume_profile / volume_profile.sum()
    per_slice = (total_shares * weights).astype(int)
    
    return per_slice

# A股典型成交量分布（以每30分钟为一段）
a_share_profile = [0.15, 0.12, 0.10, 0.09, 0.08, 
                    0.08, 0.08, 0.10, 0.12, 0.08]
schedule = vwap_schedule(100_000, a_share_profile)
# ✅ 前30分钟成交最多（15000股），午休附近最少（8000股）
```

### 6.3 回测中模拟执行（重要）

```python
def simulate_execution_with_model(price, volume, target_shares, 
                                   method='vwap', participation=0.1):
    """
    回测中更真实的执行模拟
    method: 'immediate'(立即成交) / 'twap' / 'vwap'
    participation: 参与率（成交量的占比）
    """
    if method == 'immediate':
        # 默认回测方式：立即全部成交 + 固定滑点
        slippage_cost = price * 0.001  # 0.1%滑点
        return target_shares, slippage_cost
    
    elif method == 'twap':
        # 拆成N份，每份成交都有滑点
        slices = min(10, max(1, abs(target_shares) // 1000))
        cost_per_slice = price * 0.0005  # 拆分后单次滑点变小
        total_cost = cost_per_slice * slices
        return target_shares, total_cost
    
    elif method == 'vwap':
        # 按市场成交量限制能否全部成交
        max_tradable = volume * participation
        executed = min(abs(target_shares), max_tradable)
        direction = 1 if target_shares > 0 else -1
        slippage = price * 0.0003  # VWAP滑点更小
        return executed * direction, slippage

# ✅ 方法越精细，回测越接近实盘
```
